## 5.1 硬件相关问题

### Q1: 256GB 内置硬盘真的不够用吗？

**A:** 对于你的使用场景，确实不够。

```
实际可用空间计算：
256GB 标称容量
- 20GB 系统预留
- 30GB macOS 系统
- 50GB 常用软件
= 156GB 实际可用

如果你要存储：
- 本地 AI 模型（200-500GB）❌ 不够
- 项目文件（50-100GB）❌ 不够
- 媒体文件（照片/视频）❌ 不够

所以外置 2TB 是必须的！✅
```

---

### Q2: 外置 SSD 启动会影响性能吗？

**A:** 影响很小，日常使用几乎感觉不到。

```
性能对比：
内置 SSD:    启动 25 秒，打开应用 2 秒
外置 SSD:    启动 32 秒，打开应用 2.5 秒
差距：       启动慢 7 秒，应用慢 0.5 秒

结论：完全可以接受！✅
```

---

### Q3: USB 接口速度够用吗？

**A:** 完全够用。

```
你的配置：
USB 3.2 Gen 2 = 10Gbps = 1000MB/s

实际需求：
macOS 运行：~200MB/s
模型加载：~500MB/s（峰值）
日常使用：~100MB/s

余量充足！✅
```

---

### Q4: 需要买散热配件吗？

**A:** 建议购买，但不贵。

```
推荐配件：
□ 散热支架：¥50-100（改善空气流通）
□ 导热硅胶垫：¥20（贴在 SSD 上）
□ 小风扇：¥30（对着吹）

总投资：<¥150
效果：温度降低 10-15°C
```

---

### Q5: 外置 SSD 会损坏数据吗？

```
风险因素：
```

## 5.2 系统安装问题

### Q6: 格式化 SSD 时选错磁盘怎么办？

**A:** 如果格式化了内置硬盘：

```
立即停止！不要继续操作！

解决方案：
1. 退出磁盘工具
2. 重新进入恢复模式
3. 重新安装 macOS 到内置硬盘
4. 数据可能无法恢复（新电脑应该没数据）

预防：
✓ 仔细确认磁盘容量（2TB vs 256GB）
✓ 看磁盘名称（外置通常显示品牌名）
✓ 不确定时先拍照问我
```

---

### Q7: 安装过程中断电了怎么办？

**A:** 重新开始即可。

```
步骤：
1. 重新连接电源
2. 开机进入恢复模式
3. 重新格式化 SSD（会覆盖之前的）
4. 重新安装 macOS

不会损坏硬件，放心！✅
```

---

### Q8: 安装完成后无法启动怎么办？

**A:** 按以下步骤排查：

```
排查步骤：
1. 检查 SSD 连接（重新插拔 USB 线）
2. 检查电源（确保供电稳定）
3. 进入启动选项（长按电源）
4. 手动选择 MacOS-SSD 启动
5. 如果还不行，重新安装

90% 的情况是连接问题！
```


---

### Q17: Gateway 启动失败怎么办？

**A:** 常见原因是端口被占用。

```
解决方案：
1. 检查端口占用
   lsof -i :15252

2. 如果端口被占用：
   方法 1：关闭占用进程
   kill -9 [PID]
   
   方法 2：修改端口
   编辑 openclaw.json
   "gateway": {"port": 15253}

3. 重新启动
   openclaw gateway start
```

### Q29: 如何监控系统状态？

**A:** 使用监控工具。

```bash
# 安装监控工具
brew install htop
brew install iStats

# 查看 CPU/内存
htop

# 查看温度
istats

# 查看磁盘
df -h

# 创建监控脚本
cat > ~/scripts/monitor.sh << 'EOF'
#!/bin/bash
echo "=== 系统状态 ==="
echo "CPU: $(top -l 1 | grep "CPU usage" | awk '{print $3}')"
echo "内存：$(vm_stat | awk '{print $2}' | tr -d .)"
echo "磁盘：$(df -h /Volumes/MacOS-SSD | tail -1 | awk '{print $5}')"
EOF

chmod +x ~/scripts/monitor.sh
```
